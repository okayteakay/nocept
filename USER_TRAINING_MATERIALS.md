#### **Slide 2: The Problem We're Solving (2 min)**
- **Current State (Before):**
  - Manual review of every 3-way match failure
  - AP clerk spends 15-20 minutes per exception
  - Invoices stuck in limbo for days
  - Inconsistent decisions across team
  - Email threads scattered across multiple systems

**Key Message:** This tool eliminates routine decisions and surfaces only critical cases.

#### **Slide 3: How the System Works — The Pipeline (2 min)**

**Visual:** Four-gate pipeline diagram

```
Invoice + PO + Goods Receipt
         ↓
[1] CLASSIFY & CHECK DUPLICATES
    → Detect exception type & variance
    → Check if already submitted before
    → Duplicate? ESCALATE
         ↓
[2] TOLERANCE → Is variance ≤ threshold? AUTO-APPROVE
         ↓
[3] HISTORY → Similar case approved before? AUTO-APPROVE
         ↓
[4] COMMUNICATIONS → Email/call confirms it? AUTO-APPROVE
         ↓
No gate fired? ESCALATE → Route to human reviewer
         ↓
Resolution → Memo generated → Audit trail recorded
```

**Key Metric:** System auto-approves 60-80% of exceptions. Remaining 20-40% escalate to managers with evidence.

**Key Message:** "Four deterministic gates: classify, tolerance, history, communications. No complex rules or external research—just fast, explainable decisions."

---

#### **Slide 4: What the System Does (2 min)**

**Automatic Detection:**
- Price variances (1%, 2%, 10%, etc.)
- Quantity mismatches
- Duplicate invoices (catch overpayments)
- Missing goods receipts
- Informal product substitutions
- Expedited shipping overages

**Automatic Resolution (4 gates):**
1. Within tolerance threshold → Auto-approve (confidence: 1.0)
2. Matches historical precedent → Auto-approve (confidence: 0.9)
3. Confirmed by email or call notes → Auto-approve (confidence: 0.85)
4. No variance or all gates clear → Auto-approve

**Escalation:** If no gate fires → Route to manager with evidence summary

**Key Message:** System is deterministic and conservative. It auto-rejects only duplicates. Everything else either auto-approves with evidence OR escalates for review. No external APIs or web research—pure logic.

**What the System Shows You:**

1. **Variance Summary**
   - Line-by-line breakdown (SKU, qty, price, variance %)

   - Which SKUs differ from PO

2. **Decision Rationale**
   - Which gate fired (tolerance, history, comms)
   - Confidence score (0.0–1.0)
   - Exact reasoning

3. **Communications**
   - Linked emails / call transcripts that explain the variance
   - Timestamp of communication
   - LLM analysis of relevance

4. **Historical Precedent**
   - Similar approved cases for this supplier
   - Variance range & approval dates
   - Pattern analysis

5. **Duplicate Detection**
   - If duplicate: prior exception ID
   - Timestamp of original
   - Amount, supplier, invoice number match

**How to Use It:**
- **Confidence 0.9+:** Trust the recommendation (AI is 90%+ sure)
- **Confidence 0.7–0.9:** Review the evidence (might override)
- **Confidence <0.7:** Treat as recommendation only (escalated for reason)

**Key Message:** Every decision is backed by evidence. You decide how much to trust it.

#### **Slide 11: Common Scenarios & What to Do (1.5 min)**

**Scenario 1: 1.5% Price Variance — No Evidence**
- **What Happened:** Invoice 2% higher than PO
- **System Decision:** Escalated (outside tolerance, no comms to explain)
- **Your Decision:** Check for published price increases. Supplier's market rate?
- **Action:** Approve if valid. Reject if error. Note pattern for next similar case.

**Scenario 2: Duplicate Invoice**
- **What Happened:** Same invoice number, appears twice
- **System Decision:** Auto-reject (prevents overpayment)
- **Your Action:** Verify against supplier. No further action needed.

**Scenario 3: Informal Modification (Email Confirms)**
- **What Happened:** PO qty 100, invoice qty 150, email says "customer requested extra"
- **System Decision:** Auto-approve (confidence 0.85)
- **Your Action:** Trust the system. Spot-check supplier account if volume concerns you.

**Scenario 4: Quantity Mismatch + Goods Receipt Missing**
- **What Happened:** GRN not yet received. Can't verify receipt.
- **System Decision:** Escalated (insufficient evidence)
- **Your Action:** Check receiving status. Hold approval until GRN arrives, or override if supplier is trusted.

**Key Message:** Use judgment. System gives you options; you make the call.

## Quick-Start Guide (1 Page)

### **Get Started in 5 Minutes**

**Step 1: Log In (1 min)**
- Go to: `https://nocept-system.internal/dashboard`
- Enter your email (e.g., john.doe@company.com)
- Click "Sign In" (uses company SSO)
- You're in!

**Step 2: Find Your Role (30 sec)**
- **Top-left corner:** Look for your name
- **Icon next to it:** Shows your role
  - 📋 AP Clerk (read-only)
  - ✅ AP Manager (approval authority)
  - 📊 Finance/CFO (analytics)

**Step 3: View Your First Exception (2 min)**

**If you're an AP Clerk:**
1. Click **"Exception Queue"** (left sidebar)
2. You'll see a list of all exceptions
3. Click on any exception to view details
4. Scroll down to see:
   - Variance details (price, qty, USD amount)
   - Evidence (emails, calls, research)
   - System recommendation (Auto-Approve/Reject/Escalate)
5. Back arrow to return to list

**If you're an AP Manager:**
1. Click **"Pending Approvals"** (left sidebar)
2. You'll see exceptions awaiting your decision
3. Click on any exception
4. Review the evidence (system did the hard work)
5. Click **"Approve"** or **"Reject"**
6. Optional: Add a note (e.g., "Confirmed with supplier")
7. Click **"Submit"**
   - System logs your decision
   - Audit trail updated
   - Exception marked as APPROVED/REJECTED

**If you're in Finance:**
1. Click **"Analytics"** (left sidebar)
2. You'll see:
   - **KPI Summary:** Auto-approval rate, SLA compliance, cost saved
   - **Supplier Scorecards:** Exception frequency by supplier
   - **Spend Variance:** Trends over time
3. Hover over charts for details
4. Use date filters (top-right) to zoom in/out

**Step 4: Try a Search (1 min)**

1. Go to **"Exception Queue"**
2. Look for the **search box** (top of the list)
3. Type something:
   - Invoice number (e.g., "INV-2024-001")
   - Supplier name (e.g., "Acme Corp")
   - Exception type (e.g., "Price Variance")
4. Results filter in real time
5. Click on any result to drill in

**Step 5: Understand the Colors (30 sec)**

| Color | Status | Meaning |
|-------|--------|---------|
| 🔴 Red | New/Researching | Still being processed by AI |
| 🟡 Yellow | Pending Approval | Waiting for a manager |

| 🟢 Green | Resolved | Complete, AI auto-approved |
| ✅ Blue | Approved | Complete, manager approved |
| ❌ Gray | Rejected | Complete, manager rejected |

#### **Section 5: What Confidence Scores Mean**

**System shows a "Confidence" score for each recommendation (0.0 to 1.0):**

- **0.9–1.0 (High Confidence):** AI is 90%+ certain. Evidence is strong. Manager likely to approve.
- **0.75–0.89 (Good Confidence):** AI found supporting evidence (comms, history, research). Reasonable to approve.
- **0.5–0.74 (Moderate Confidence):** Some evidence, but not conclusive. Manager should review.
- **<0.5 (Low Confidence):** Minimal evidence. Manager must decide. Flag for escalation.

**How You Can Use This:**
- High confidence exceptions often auto-approve by manager. Less risky.
- Low confidence exceptions need extra scrutiny. Ask manager to call supplier if unsure.

#### **Section 6: Red Flags to Watch For**

**Flag for Your Manager If You See:**

1. **Duplicate Invoice** ❌
   - Same invoice number appearing twice
   - System auto-rejects (prevents overpayment)
   - But verify with supplier—might be our error

2. **Unexplained High Variance** 🚩
   - >10% variance with no comms or history
   - Could be supplier error or intentional change
   - Ask manager to contact supplier

3. **Serial Supplier Issues** 🚩
   - Same supplier has >3 exceptions in a week
   - Pattern suggests quality control issue on their end
   - Escalate to procurement for vendor review

4. **Missing Goods Receipt (GRN)** 📦
   - Item shows on invoice but no receipt yet
   - Can't verify if goods arrived
   - Hold approval until GRN received (check receiving log)

5. **Communication Evidence But Low Confidence** 🤔
   - Email or call confirms exception, but AI confidence <0.75
   - Example: Email is vague or written by non-authorized person
   - Manager should call supplier directly to confirm

#### **Section 3: Creating and Managing Rules**

**What Are Rules?**

Rules let you **automate future decisions** for recurring patterns.

**Example Rules:**

| Rule | Condition | Action |
|------|-----------|--------|
| Acme Price Variance | Supplier=Acme, Variance 2–3%, Email confirms | Auto-approve |
| High-Value Orders | Amount >$50k, Variance <5% | Escalate to CFO instead of you |
| Duplicate Detection | Exact invoice match, same vendor | Auto-reject |
| Known Suppliers | Supplier in "Trusted" list, any variance <10% | Auto-approve |

**To Create a Rule:**

**Method 1: From an Exception**
1. Open exception you want to turn into a rule
2. Click **"Create Rule"** (if applicable)
3. System pre-fills supplier, exception type, variance range
4. Click **"Save Rule"**

**Method 2: Manual Rule Creation**
1. Click **"Rules"** (left sidebar)
2. Click **"Create New Rule"** (top-right)
3. Fill out form:
   - **Rule Name:** (e.g., "Acme Corp Auto-Approval")
   - **Supplier:** (e.g., "Acme Corp") or leave blank for all
   - **Exception Types:** Price Variance, Qty Mismatch, etc.
   - **Variance Range:** Min % to Max %
   - **Additional Conditions:** (e.g., "Require email confirmation," "Limit to <$10k")
   - **Action:** Auto-Approve / Auto-Reject / Escalate
   - **Confidence Threshold:** (0.0–1.0; system must be at least this confident)
4. Click **"Save Rule"**

**To Manage Rules:**

1. Click **"Rules"** (sidebar)
2. View all rules you've created
3. For each rule:
   - **Edit:** Adjust variance range, conditions, action
   - **View Usage:** See how many times this rule auto-approved (count, dates)
   - **Disable:** Turn off without deleting (useful for testing)
   - **Delete:** Remove permanently

**Rule Best Practices:**

- Start narrow (single supplier, narrow variance range). Broaden as you gain confidence.
- Include "max amount" limits. (E.g., auto-approve Acme variances only if <$5k.)
- Use "require comms confirmation" if pattern depends on email evidence.
- Review rule usage monthly. If 100+ auto-approvals with zero rejections, consider broadening.
- Disable rules if supplier behavior changes (e.g., new contact, quality issues).

#### **Section 4: Interpreting Evidence**

**Confidence Score (0.0–1.0):**

| Score | Meaning | What You Should Do |
|-------|---------|-------------------|
| 0.9–1.0 | AI is 90%+ certain | Trust it. Approve with confidence. |
| 0.75–0.89 | Good supporting evidence | Approve. Spot-check occasionally. |
| 0.5–0.74 | Some evidence, but mixed | Review carefully. Call supplier if unsure. |
| <0.5 | Weak evidence | Reject or ask for more info. |

**What Each Evidence Type Means:**

**1. Tolerance Gate (Variance ≤ 1%)**
- System automatically approved
- Variance is within normal rounding/market fluctuation
- No manager review needed
- You'll rarely see these (already resolved)

**2. Historical Precedent (Similar approved cases)**
- System found past cases like this
- Same supplier, similar variance %, approved before
- Suggests pattern is legitimate
- Confidence: 0.85–0.90
- Your action: Approve unless supplier circumstances changed

**3. Communications (Email/Call confirms variance)**
- Linked email or call transcript supports the variance
- Example: Email says "Customer requested 50 extra units"
- Confidence depends on:
  - Who sent the email? (Authorized contact?)
  - How specific? (Vague vs. detailed?)
  - Timestamp? (Before or after invoice date?)
- Your action: High confidence (0.85+) = approve. Medium (0.7–0.84) = call supplier.

**4. Research (Web search corroborates)**
- System found public source supporting exception
- Example: Supplier's website shows current price = invoice price (which is higher than PO)
- Confidence: 0.75–0.80
- Your action: Approve, but note it (evidence can change)

**5. No Gate Fired → Escalated**
- Variance is real, but no evidence supports it
- Could be legitimate (supplier error) or fraudulent (overpayment attempt)
- System can't decide, routed to you
- Your action: Call supplier. Ask why variance exists. Decide based on conversation.

#### **Section 5: Red Flags — When to Reject**

**Reject If Any of These Are True:**

1. **Variance ≥10% with No Evidence**
   - Large variance, no comms/research/history support it
   - Likely supplier error or intentional overpayment
   - Action: Reject, contact supplier to resubmit

2. **Communication Is From Unauthorized Party**
   - Email from supplier's salesman (not procurement), suggesting informal deal
   - Example: "Just between us, add 50 units at the regular price"
   - Red flag: Circumventing PO process
   - Action: Reject, escalate to procurement

3. **Goods Receipt Still Missing (No GRN)**
   - Can't verify goods arrived
   - Can't determine if quantity variance is legitimate
   - Action: Reject, ask receiving when GRN will arrive

4. **Duplicate Invoice (AI Auto-Rejects)**
   - Same invoice number from same supplier
   - Likely our posting error, not supplier's fault
   - Action: System auto-rejects; you verify with accounting

5. **Supplier on "Watch List"**
   - This supplier has had 3+ variance issues this quarter
   - Pattern suggests quality control issue
   - Action: Reject, escalate to procurement for vendor review

6. **Price Variance with No Comms but High Confidence**
   - System claims high confidence (0.9+) but you don't see evidence
   - Could be system error or missing data
   - Action: Reject with note "Unable to verify evidence," ask system support

#### **Section 2: Interpreting KPI Dashboard**

**Metric 1: Total Exceptions**

**Metric 7: Cost Saved ($)**
- Sum of overpayments prevented (e.g., duplicate invoices rejected)
- Calculation: Variances on rejected exceptions = money we didn't lose
- ROI: Cost Saved / System Cost = payback period
- Example: System costs $10k/year, saves $100k in prevented overpayments = 10x ROI

#### **Section 4: Spend Variance Analysis**

**What Is Spend Variance?**

The difference between what we agreed to pay (PO) and what we were invoiced.

```
Spend Variance = Invoice Total - PO Total
Variance % = (Variance / PO Total) × 100%
```

**Example:**
- PO total: $100,000
- Invoice total: $102,500
- Variance: +$2,500 (supplier over-charged)
- Variance %: 2.5%

**Spend Variance Report: What You'll See**

```

Variance by Exception Type:
  - Price Variance: $47,300 (52%)
  - Qty Mismatch: $23,100 (25%)
  - Expedited Shipping: $12,500 (14%)
  - Other: $8,200 (9%)
  TOTAL: $91,100

Variance by Supplier (Top 10):
  1. Acme Corp: +$12,300 (mostly price)
  2. Widget Inc: +$8,900 (qty & price)
  3. Office Depot: +$6,700 (expedited shipping)
  ...

Variance Trend (Last 6 Months):
  Jan: $18,000
  Feb: $21,000
  Mar: $19,500
  Apr: $22,300
  May: $23,100
  TREND: Slightly increasing

Financial Impact:
  - Total variances before approval: $91,100
  - Variances approved: $68,300
  - Variances rejected: $22,800
```

**How to Interpret:**

   - Alternate suppliers if variance grows

3. **Variance Trend:** Getting better or worse?
   - Increasing = suppliers are more aggressive, or market inflation
   - Decreasing = better processes, or supplier relationships improving
   - Compare to external benchmarks (CPI inflation, market rates)

4. **Cost Saved (Rejections):** ROI of system
   - Example: Rejected $22,800 in bad exceptions = system paying for itself

### **Video 2: Creating Rules (2 minutes)**

**[0:00–0:15] Intro**

"In this video, we'll show how to create a rule. Rules let you automate future approvals. Once you create a rule, similar exceptions auto-approve without waiting for you."

**[0:15–0:30] What Are Rules?**

"Let's say you approve the same type of exception repeatedly. Example: Acme Corp always has 2–3% price variances, and each time you approve it because you've negotiated that with them.

Instead of approving manually each time, create a rule: 'If supplier = Acme Corp AND variance = 2–3% AND amount <$10k, auto-approve.'

From that point on, all matching exceptions auto-approve automatically. Saves you ~5 minutes per exception."

**[0:30–1:00] Creating a Rule (Method 1: From Exception)**

"Here's the fast way. Open an exception you want to turn into a rule. Let's say you just approved an Acme Corp exception.

[Screen: Open Acme Corp exception]

At the bottom, click 'Create Rule from This Case.' The system pre-fills the supplier, exception type, and variance range. You can adjust if needed.

[Screen: Click button, form appears]

Check the conditions: Supplier = Acme Corp, Exception Type = Price Variance, Variance Range = 2% to 3%. Looks good. Set 'Max Amount' to $10k. Click 'Save Rule.'

Done. From now on, Acme Corp price variances 2–3% under $10k auto-approve."

**[1:00–1:30] Creating a Rule (Method 2: Manual)**

"Alternatively, click 'Rules' in the sidebar, then 'Create New Rule.' You fill out a form:

[Screen: Show rules form]

- Rule Name: e.g., 'Acme Corp Auto-Approval'
- Supplier: e.g., 'Acme Corp' (or leave blank for all)
- Exception Type: Price Variance, Qty Mismatch, etc.
- Variance Range: Min % to Max %
- Max Amount: e.g., $10,000
- Additional Conditions: Require email confirmation, etc.
- Action: Auto-Approve / Auto-Reject / Escalate

Fill it out and click 'Save.' System remembers this rule."

**[1:30–2:00] Managing Rules**

"You can see all your rules by clicking 'Rules' in the sidebar.

[Screen: Show rules list]

For each rule, you can:
- **Edit:** Adjust conditions (e.g., increase max amount)
- **View Usage:** See how many times this rule fired (e.g., '47 auto-approvals in May')
- **Disable:** Turn it off temporarily without deleting
- **Delete:** Remove permanently

That's rules. Use them to automate your most common approvals. Questions? See the user manual."

**[2:00] Outro**

### **Video 4: Common Scenarios (4 minutes)**

**[0:00–0:20] Intro**

"In this video, we walk through four real-world scenarios. Each shows what the system displays and what you should do."

**[0:20–1:00] Scenario 1: Price Variance, No Evidence**

"You get an exception: Acme Corp invoice shows $105 per unit, but PO shows $100. That's a 5% price variance.

[Screen: Show exception detail]

System says: 'Escalated. Variance is above 1% tolerance. No comms or research found. Confidence: 0.42.'

What to do: This is a real variance, but system can't justify it. You need to decide. Call Acme Corp's procurement contact: 'I see your invoice is 5% higher than PO. Did we discuss this?' Their answer determines your decision.

If they say, 'Yes, market rate increased,' approve it and create a rule. If they say, 'We don't know, was it an error?' tell them to resubmit with correct price."

**[1:00–1:40] Scenario 2: Duplicate Invoice**

"You see an exception: Same invoice number (INV-20245678) appearing twice from Acme Corp, 3 days apart.

[Screen: Show exception detail]

System says: 'Auto-rejected. Duplicate detected. Confidence: 1.0.'

What to do: System is correct. But you need to investigate: Did we post it twice? Or did Acme send it twice? Check:
1. Accounting system: Search invoice number, see if posted twice
2. Acme email: Did we receive two emails or one?

If we posted twice, reverse one posting immediately (prevents overpayment). If they sent twice, tell Acme to resubmit the correct one.

Cost saved: This exception cost you zero (caught by system)."

**[1:40–2:20] Scenario 3: Qty Mismatch, Email Confirms**

"You see an exception: PO qty 100 units, invoice qty 150 units (50% variance).

[Screen: Show exception detail]

But system found a linked email from May 15: '[Email quote] Hi John, customer called today asking for 50 extra units ASAP. Can you add 50 to the purchase order we discussed? We'll invoice you separately.' —Acme sales contact

System says: 'Auto-approved. Email confirms customer-requested extra. Confidence: 0.88.'

What to do: Trust the system. Confidence is high because email evidence is clear. Approve it (already auto-approved, but you spot-check). Create a rule: 'If supplier = Acme, exception type = qty mismatch, email confirms customer request, auto-approve up to $5k.'

Future similar exceptions auto-approve."

**[2:20–3:00] Scenario 4: Missing Goods Receipt (GRN)**

"You see an exception: Invoice shows delivery of 200 units on May 20. But GRN (goods receipt) hasn't been posted yet.

[Screen: Show exception detail]

System says: 'Escalated. Goods receipt missing. Can't verify delivery. Confidence: 0.35.'

What to do: Can't decide without GRN. Check receiving status: Call receiving dept: 'Did we receive 200 units from Acme on May 20?' 

If yes: 'Why isn't GRN posted?' Get ETA for GRN posting. Hold approval until GRN arrives.

If no: 'When will it arrive?' Supplier shipped but goods in transit. Hold approval.

If we reject now, supplier might reship, causing duplicate. Better to wait for GRN."

**[3:00–3:30] What These Scenarios Show**

"Notice the pattern:

1. **No Evidence → Use Judgment** (call supplier)
2. **Duplicate → Prevent Fraud** (stop overpayment)
3. **Comms Confirm → Trust AI** (approve)
4. **Missing Data → Hold Decision** (wait for completion)

The system gives you all the info. You make the final call. That's the partnership: AI finds patterns and evidence, you use judgment and experience."

**[3:30–4:00] Outro**

"These scenarios are real. You'll see all four types this week. The key: Read the evidence, use your knowledge of suppliers and processes, and decide confidently. Questions? See the user manual or email support."

## Appendix: System Glossary

**Glossary of Terms**

| Term | Definition |
|------|------------|
| **Exception** | A three-way match failure: invoice vs. PO vs. goods receipt don't align. |
| **Variance** | The difference between invoice amount and PO amount. Measured in USD or %. |
| **Three-Way Match** | Verification that invoice, PO, and goods receipt all agree on qty, price, date. |
| **Auto-Approve** | System makes approval decision without human review (confidence high). |
| **Escalate** | System can't decide; routes to human manager for review. |
| **GRN (Goods Receipt Note)** | Record of goods received from supplier (warehouse/receiving system). |
| **Confidence Score** | System's certainty in recommendation (0.0–1.0; 0.9+ is high). |
| **Rule** | User-defined automation: if conditions match, auto-approve/reject. |
| **Audit Trail** | Complete history of all decisions, approvals, rejections, changes. |
| **Supplier Scorecard** | Summary of one supplier's exception frequency, types, trends. |
| **Spend Variance** | Total $ difference between all invoices and their POs (for analysis). |
| **SLA (Service Level Agreement)** | Target resolution time (e.g., resolve 95% of exceptions <4 hours). |
