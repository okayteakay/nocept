# Dataset

Synthetic AP (Accounts Payable) dataset for **Meridian Corp** — used to train/test the Invoice Exception Resolution Agent.

Generated by `generate_data.py`. Schema defined in `models.py`. Catalog metadata in `catalog.json`.

---

## Files (`data/`)

| File | Records | Description |
|------|---------|-------------|
| `invoices.json` | 200 | Invoices with line items (SKU, qty, unit price, totals) |
| `purchase_orders.json` | 200 | POs with line items; linked to invoices via `po_number` |
| `goods_receipts.json` | 190 | GRNs confirming delivery; linked to PO + invoice |
| `exception_records.json` | 70 | Flagged discrepancies with variance amounts and linked comms |
| `emails.json` | 100 | Supplier/buyer email threads referencing POs and invoices |
| `phone_transcripts.json` | 40 | Call transcripts between suppliers and buyers |
| `suppliers.json` | 12 | Supplier master: contact info, product catalog, pricing |

---

## Key Relationships

```
purchase_orders ──< goods_receipts
       │                  │
       └──────┬───────────┘
           invoices ──< exception_records ──> emails / phone_transcripts
```

Join key across all documents: `po_number`. Invoices also carry `invoice_number` / `supplier_id`.

---

## Exception Types

| Type | Meaning |
|------|---------|
| `price_variance` | Invoice unit price differs from PO |
| `quantity_variance` | Invoiced qty doesn't match GRN qty |
| `informal_modification` | SKU/grade substituted without a formal PO amendment |
| `missing_goods_receipt` | No GRN exists for the invoice |
| `duplicate_invoice` | Same PO billed more than once |

---

## Line Item Fields

`invoices.json`, `purchase_orders.json`, `goods_receipts.json` all share nested `line_items`:

| Field | Notes |
|-------|-------|
| `sku` | Supplier SKU |
| `description` | Product name |
| `product_grade` | Standard / Premium / Ultra |
| `unit_price` | Per-unit cost |
| `quantity` | Units ordered/invoiced/received |
| `total` | `unit_price × quantity` |
